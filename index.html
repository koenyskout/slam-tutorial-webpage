Overview

The SLAM Recursion: Predict, Correct, Repeat

At each time step, SLAM combines a motion model (prediction) with sensor data (correction). This repeats continuously while the robot moves. The result is a belief over both robot state and map.

\[ p(x_k,m \mid z_{1:k},u_{1:k}) \propto p(z_k \mid x_k,m) \int p(x_k \mid x_{k-1},u_k)\,p(x_{k-1},m \mid z_{1:k-1},u_{1:k-1})\,dx_{k-1} \]

Sections 1-5 build intuition for each term in this recursion. Sections 6-8 combine them into full online mapping and physically plausible state estimation.

Section 1

State, Frames, and Coordinate Transforms

SLAM begins with a state definition. In 2D, robot pose is typically represented as \(\mathbf{x} = (x, y, \theta)\), where \(x,y\) are position in a world frame and \(\theta\) is heading. Sensors often measure points in the robot's local frame, not in world coordinates.

A local point \((p_x, p_y)\) converts to world coordinates by rigid transform:

\[ \begin{aligned} x_w &= x + p_x\cos\theta - p_y\sin\theta \\ y_w &= y + p_x\sin\theta + p_y\cos\theta \end{aligned} \]

Move the robot and local point in the figure. This is the geometric core underneath many SLAM measurement models.

Why this matters. Most SLAM errors are ultimately transform or frame-consistency errors.

Drag the robot, heading handle, or transformed point directly on the figure.

Section 2

Prediction from Odometry

In dead reckoning, pose is updated only from motion increments. If the robot moves by \(\Delta s\) and rotates by \(\Delta\theta\), a common model is:

\[ \begin{aligned} x_k &= x_{k-1} + \Delta s \cos(\theta_{k-1}) \\ y_k &= y_{k-1} + \Delta s \sin(\theta_{k-1}) \\ \theta_k &= \theta_{k-1} + \Delta\theta \end{aligned} \]

Because odometry increments are noisy, drift grows with time and distance. This section shows how fast prediction can diverge without external correction.

Core limitation. Prediction-only localization is unbounded in error over long trajectories.

Section 3

Probabilistic Fusion: Prior + Measurement = Posterior

SLAM is probabilistic. Instead of one exact pose, we keep a belief distribution. The prior belief from prediction is fused with a measurement likelihood to produce a posterior.

In 1D Gaussian form, this update has closed-form solution. Let prior be \(\mathcal{N}(\mu_p, \sigma_p^2)\) and measurement be \(\mathcal{N}(z, \sigma_z^2)\). Then:

\[ K = \frac{\sigma_p^2}{\sigma_p^2 + \sigma_z^2},\qquad \mu_{post} = \mu_p + K(z - \mu_p),\qquad \sigma_{post}^2 = (1-K)\sigma_p^2 \]

The posterior is narrower (more certain) than either source if both are informative. This is the algebraic intuition behind Kalman-style SLAM updates.

Drag the peak markers to move means. Drag sigma handles (small circles on each curve) to change uncertainty.

Section 4

Measurement Update with Known Landmarks

Prediction is corrected by comparing actual measurements to expected measurements. In compact notation:

\[ \begin{aligned} \hat{x}_k^- &= f(\hat{x}_{k-1}, u_k) \\ \hat{x}_k &= \hat{x}_k^- + K_k\left(z_k - h(\hat{x}_k^-)\right) \end{aligned} \]

Here landmarks are assumed known. The corrected estimate is continuously pulled back toward geometric consistency with observed landmarks.

Tuning insight. Gain trades responsiveness for noise sensitivity.

Section 5

Data Association: Matching Measurements to Landmarks

A measurement is only useful if we know which landmark produced it. This is the data association problem. In cluttered scenes, nearby landmarks can produce ambiguous matches.

The simplified figure samples noisy range-bearing measurements from one true landmark, then computes relative likelihood for each candidate landmark. If separation is small and noise is high, wrong matches increase.

\[ p(z\mid \ell_i) \propto \exp\left(-\frac{1}{2}\Big[\big(\tfrac{\Delta r_i}{\sigma_r}\big)^2 + \big(\tfrac{\Delta \phi_i}{\sigma_\phi}\big)^2\Big]\right) \]
Failure mode. Wrong associations can corrupt both map and trajectory even when other components are correct.

Section 6

Online SLAM: Estimating Pose and Map Together

Full SLAM estimates unknown landmarks and robot pose simultaneously. The estimates are coupled: better pose gives better map updates, and better map gives better localization corrections.

\[ p(x_{0:k}, m \mid z_{1:k}, u_{1:k}) \]

This joint posterior (trajectory \(x_{0:k}\) and map \(m\)) is the central SLAM object. The visualization uses a simplified online variant with landmark refinement and loop-closure-like corrections.

Modeling simplification. This teaching demo assumes known landmark identities after association and uses lightweight correction rules instead of full sparse graph optimization or full EKF covariance propagation.

Section 7

Occupancy Grid Mapping from Range Beams

Not all SLAM maps are sparse landmarks. A common alternative is occupancy grids, where each cell stores probability of being occupied. Range beams update free space along the ray and occupied space at the hit.

\[ L_t(c) = L_{t-1}(c) + \begin{cases} +\lambda_{occ}, & \text{if beam hits cell } c \\ -\lambda_{free}, & \text{if beam passes through } c \end{cases} \]
\[ p_t(c)=\sigma(L_t(c))=\frac{1}{1+e^{-L_t(c)}}, \qquad \mathcal{F}=\{c:\,L_t(c)

Section 8

Kinematic Constraints: Preventing Non-Physical Pose Jumps

If corrections are applied directly to pose, estimates can jump unrealistically. For physical robots, state updates should respect motion constraints.

\[ \dot{x}=v\cos\theta,\quad \dot{y}=v\sin\theta,\quad \dot{\theta}=\omega,\quad |\omega|\le \omega_{\max},\; |\dot{v}|\le a_{\max} \]

The green estimate applies bounded yaw-rate and acceleration. The dashed gray baseline applies unconstrained corrections. Compare jump metrics to see why motion priors improve physical plausibility.
