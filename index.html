Lesson 1

Dead Reckoning and Why Drift Is Inevitable

Odometry predicts movement from wheel encoders and gyros. It is fast and continuous, but every tiny error is integrated, so uncertainty keeps growing.

Learning Goals

  • Understand prediction-only localization.
  • See cumulative drift as a function of noise and distance.
  • Read pose error directly from true vs estimated tracks.

Try This

Set odometry noise above 1.8 and watch how quickly drift increases over 20 seconds.

Lesson 2

Measurement Update: Landmarks Correct Drift

SLAM fuses a prediction (odometry) with observations (landmarks). This update keeps the state estimate anchored to physical measurements.

Learning Goals

  • Understand prediction vs correction phases.
  • Explore gain tuning and noise tradeoffs.
  • Observe corrected error relative to raw odometry error.

Try This

Raise sensor noise above 1.6, then increase correction gain. Notice over/under correction behavior.

Lesson 3

Online SLAM: Build the Map While Estimating Pose

Unknown landmarks are initialized from observations, then refined every time the robot revisits them. This couples localization and mapping.

Learning Goals

  • Understand why map quality improves with revisits.
  • Observe uncertainty shrinking from repeated measurements.
  • Relate loop closures to global consistency.

Try This

Run until at least 20 landmarks are mapped. Toggle ground truth to compare estimated map quality.

Lesson 4

Kinematic Constraints: Physically Plausible SLAM

State updates should respect vehicle dynamics. By enforcing acceleration and yaw-rate limits, correction steps become smooth and physically realistic.

Learning Goals

  • See how unconstrained correction creates jumpy trajectories.
  • Use kinematic limits to smooth updates.
  • Compare raw and constrained tracking errors.

Try This

Enable baseline, raise correction gain, and lower max yaw-rate. Compare jump metrics in the readout.
